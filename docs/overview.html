서비스 컨셉

물리적인 포스트잇처럼 화면 어디에나 메모를 붙이고, 자유롭게 이동할 수 있는 웹 애플리케이션. 별도 계정이나 서버 없이 브라우저만으로 동작하며, 새로고침 후에도 메모가 유지된다.

핵심 사용 시나리오캔버스를 더블클릭해 메모를 생성하고, 툴바를 드래그해 원하는 위치로 이동한다. 색상을 바꿔 메모를 분류하고, 작업이 끝나면 삭제한다. 모든 변경은 자동으로 저장된다.

기획 목표

🎯

즉각적인 조작감

더블클릭 → 생성, 드래그 → 이동. 별도 버튼 없이 직관적으로 동작한다.

💾

자동 저장

저장 버튼 없음. 모든 변경이 즉시 localStorage에 반영된다.

🚀

빠른 응답성

드래그 중 스토어를 건드리지 않아 60fps 부드러운 이동이 가능하다.

🔌

백엔드 없음

순수 프론트엔드. 서버 없이 어디서나 실행 가능하다.

🧩

확장 가능한 구조

훅·컴포넌트·스토어가 분리되어 기능 추가가 쉽다.

♿

접근성

aria 속성 지원, 키보드 포커스 관리, 스크린리더 레이블 제공.

z-index 관리 방식

배열 순서 = z-index숫자로 z-index를 직접 관리하면 메모가 많아질수록 renumbering이 필요하다. 대신 memos 배열의 마지막 원소가 가장 높은 z-index를 가지도록 설계했다. 클릭 시 reorderToTop이 해당 메모를 배열 맨 끝으로 splice한다.

이벤트 캡처 단계 사용메모 클릭은 onPointerDownCapture(캡처 단계)로 처리한다. MemoContent 에디터 영역의 stopPropagation(버블 단계)이 있어도 영향받지 않으므로 텍스트 영역 클릭 시에도 z-index가 정상적으로 갱신된다.

상태 변경 + 저장

컴포넌트는 스토어를 직접 변경하지 않고 반드시 액션을 통해 변경한다.

원칙적용 방식
관심사 분리 타입(types) · 상태(store) · 로직(hooks) · UI(components) · 저장(utils)을 계층으로 분리
단일 책임 각 컴포넌트·훅은 하나의 역할만 담당. MemoContent는 편집만, MemoToolbar는 설정 UI만.
성능 최적화 드래그 중 스토어 쓰기 없음 (CSS transform만). onBlur에서만 content 저장. subscribe로 렌더 외부 저장.
타입 안전성 모든 타입을 types/memo.ts 한 곳에서 관리. MemoColor 유니온으로 오타 컴파일 오류 처리.
확장 가능성 PersistedState.version으로 스키마 마이그레이션 지원. 지도 저장은 data 속성 직렬화로 구조 변경 없이 확장 가능.

저장 전략

저장 위치

브라우저 localStorage 전용 서버나 외부 DB 없이 브라우저 로컬 스토리지에만 저장된다. 같은 브라우저·같은 도메인 내에서만 유지되며, 시크릿 모드에서는 탭 종료 시 삭제된다. 브라우저의 "사이트 데이터 삭제" 또는 localStorage.clear()로 영구 삭제된다.

항목값
스토리지 키memo-app-v1
저장 형식{ "memos": [...], "version": 1 }
용량 제한브라우저 기본 (~5MB). 초과 시 try-catch로 조용히 무시.
시크릿 모드탭 종료 시 삭제. 앱은 계속 동작.

저장 시점 (자동 저장)

저장 버튼 없음 — 모든 변경이 즉시 자동 저장 Zustand subscribe()로 React 렌더 사이클 바깥에서 memos 변경을 감지한다. 참조 비교(state.memos !== prevMemos)로 실제 변경이 있을 때만 localStorage.setItem을 호출해 불필요한 쓰기를 최소화한다.

스키마 마이그레이션

현재 버전: CURRENT_VERSION = 1 PersistedState.version 필드로 관리한다. Memo 구조가 바뀌면 CURRENT_VERSION을 올리고 localStorage.ts의 migrate() 함수에 버전별 변환 로직을 추가한다. 이를 통해 기존 사용자의 데이터를 잃지 않고 안전하게 업그레이드할 수 있다.
