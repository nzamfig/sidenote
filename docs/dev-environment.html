테스트 설정

src/test/setup.ts
import '@testing-library/jest-dom';
// toBeInTheDocument(), toHaveTextContent() 등 DOM matcher 전역 등록

테스트 환경 구성 방식 Vitest는 vite.config.ts의 test.environment: 'jsdom' 설정으로 브라우저 DOM을 에뮬레이션한다. test.globals: true로 describe, it, expect 등을 import 없이 전역으로 사용한다. localStorage는 jsdom이 자동으로 제공하므로 별도 모킹이 필요 없다.

알려진 제약

Zustand v5 subscribe API 변경 Zustand v4까지는 subscribe(selector, callback) 형태로 선택적 구독이 가능했다. v5에서는 subscribe(listener) 단일 콜백만 지원하므로, usePersistence.ts에서 이전 참조(prevMemos)와 비교해 실제 변경 시에만 저장하도록 구현됐다.

Node.js 버전에 따른 패키지 제약 vitest v4.x와 jsdom v29.x는 Node v20.12.0 이상이 필요하다. 현재 환경(Node v20.3.0)에서는 각각 v1.x, v24.x를 사용한다. 기능 차이는 없으나 Node 업그레이드 후 npm install을 다시 실행하면 최신 버전으로 전환할 수 있다.

드래그 & 드롭 테스트 @dnd-kit은 실제 PointerEvent가 필요해 jsdom 환경에서 완전한 드래그 시뮬레이션이 어렵다. 드래그 동작 통합 테스트는 Playwright 또는 Cypress 같은 E2E 테스트 도구로 작성하는 것이 권장된다.

시퀀스 다이어그램 파일 (docs/sequence-diagrams.html) Mermaid를 CDN에서 로드하므로 인터넷 연결이 필요하다. 오프라인 환경에서는 Node v20.12.0 이상에서 npx @mermaid-js/mermaid-cli로 PNG 변환 가능.
