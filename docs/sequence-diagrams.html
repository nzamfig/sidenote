1
앱 시작 — localStorage 복원 (Hydration)
페이지 로드 시 저장된 메모를 스토어에 주입하고 자동 저장 감시를 시작한다
sequenceDiagram participant Browser as 브라우저 participant App participant usePersistence as usePersistence 훅 participant LS as localStorage 유틸 participant Store as Zustand 스토어 participant Canvas Browser->>App: 페이지 로드 App->>usePersistence: usePersistence() 호출 App->>Canvas: <Canvas /> 렌더링 (memos: []) Note over usePersistence: useEffect 마운트 실행 usePersistence->>LS: loadMemos() LS->>LS: getItem('memo-app-v1') alt 저장된 데이터 있음 LS->>LS: JSON.parse() + migrate() LS-->>usePersistence: Memo[] usePersistence->>Store: hydrateFromStorage(memos) Store-->>Canvas: memos 업데이트 → 리렌더 else 최초 실행 (데이터 없음) LS-->>usePersistence: [] Note over usePersistence: 아무것도 하지 않음 end usePersistence->>Store: subscribe() 등록 (자동 저장 감시 시작)
2
메모 생성 — 캔버스 더블클릭
클릭 좌표를 캔버스 기준으로 변환해 새 메모를 생성하고 즉시 저장한다
sequenceDiagram actor User as 사용자 participant Canvas participant Interaction as useCanvasInteraction 훅 participant Store as Zustand 스토어 participant LS as localStorage 유틸 participant Memo User->>Canvas: 캔버스 배경 더블클릭 Canvas->>Interaction: handleDoubleClick(e) Interaction->>Interaction: closest('[data-memo-id]') 확인 Note over Interaction: 기존 메모 위가 아니면 계속 진행 Interaction->>Interaction: getBoundingClientRect()로
캔버스 기준 좌표 계산
x = clientX - rect.left
y = clientY - rect.top Interaction->>Store: createMemo({ position: {x, y} }) Store->>Store: generateId()로 id 생성 Store->>Store: memos 배열 맨 끝에 새 Memo 추가 Store->>Store: activeMemoId = 새 메모 id Store-->>Canvas: memos 변경 → 리렌더 Canvas->>Memo: <Memo memo={newMemo} zIndex={N} /> Memo->>Memo: isNew = true → autoFocus 전달 Note over Memo: textarea에 자동 포커스
바로 타이핑 가능 Store-->>LS: subscribe 콜백 → saveMemos() LS->>LS: setItem('memo-app-v1', JSON)
3
메모 드래그 이동
드래그 중에는 CSS transform만 변경하고, 드래그 종료 시점에만 스토어에 위치를 커밋한다
sequenceDiagram actor User as 사용자 participant Toolbar as MemoToolbar (드래그 핸들) participant DndKit as @dnd-kit/core participant Memo participant Canvas participant Store as Zustand 스토어 participant LS as localStorage 유틸 User->>Toolbar: 포인터 누름 (pointerdown) Toolbar->>Memo: onPointerDown 실행 Memo->>Store: reorderToTop(id) Store->>Store: 해당 메모를 배열 맨 끝으로 이동
→ z-index 최상위 Memo->>Store: setActiveMemo(id) Toolbar->>DndKit: listeners.onPointerDown 전달 DndKit->>DndKit: PointerSensor: 5px 이상 이동 대기 User->>Toolbar: 포인터 5px 이상 이동 DndKit->>DndKit: 드래그 시작 (isDragging = true) DndKit-->>Memo: transform = {x: dx, y: dy} 전달 loop 포인터 이동 중 (매 프레임) User->>Toolbar: 포인터 이동 DndKit-->>Memo: transform 업데이트 Memo->>Memo: style.transform = translate(dx, dy)
left/top은 변경 없음 Note over Memo: GPU transform으로 렌더링
스토어·localStorage 쓰기 없음 end User->>Toolbar: 포인터 놓음 (pointerup) DndKit->>Canvas: onDragEnd({ active, delta }) Canvas->>Canvas: rawX = position.x + delta.x
rawY = position.y + delta.y Canvas->>Canvas: 경계 클램핑
x = clamp(rawX, 0, canvasW - memoW)
y = clamp(rawY, 0, canvasH - memoH) Canvas->>Store: moveMemo(id, { x, y }) Store->>Store: position 업데이트, updatedAt 갱신 Store-->>Memo: transform = null → left/top으로 확정 Store-->>LS: subscribe 콜백 → saveMemos()
4
메모 내용 편집
입력 중에는 React·스토어가 관여하지 않고, 포커스를 잃는 순간(onBlur)에만 저장한다
sequenceDiagram actor User as 사용자 participant MemoContent participant Store as Zustand 스토어 participant LS as localStorage 유틸 Note over MemoContent: textarea는 비제어 컴포넌트
(defaultValue 사용, React가 값 추적 안 함) User->>MemoContent: textarea 클릭 (포커스 획득) MemoContent->>MemoContent: onPointerDown → stopPropagation()
드래그 핸들로 이벤트 전파 차단 loop 텍스트 입력 중 User->>MemoContent: 키 입력 Note over MemoContent: DOM이 값을 직접 관리
리렌더 발생하지 않음 end User->>MemoContent: 다른 영역 클릭 (onBlur) MemoContent->>MemoContent: e.target.value !== content 비교 alt 내용이 변경됐을 때 MemoContent->>Store: onChange(newValue)
→ updateMemo(id, { content }) Store->>Store: content 업데이트, updatedAt 갱신 Store-->>LS: subscribe 콜백 → saveMemos() else 내용이 변경되지 않았을 때 Note over MemoContent: 스토어·localStorage 쓰기 생략 end

색상 변경 및 메모 삭제
툴바의 두 가지 액션. 모두 스토어를 통해 처리되고 localStorage에 즉시 반영된다
sequenceDiagram actor User as 사용자 participant Toolbar as MemoToolbar participant ColorPicker participant Actions as useMemoActions 훅 participant Store as Zustand 스토어 participant LS as localStorage 유틸 rect rgb(237, 246, 255) Note over User, LS: 색상 변경 흐름 User->>Toolbar: 색상 스와치 클릭 Toolbar->>ColorPicker: onChange(color) 전달 ColorPicker->>Actions: onColorChange(color) Actions->>Store: updateMemo(id, { color }) Store->>Store: color 업데이트, updatedAt 갱신 Store-->>Toolbar: 리렌더 → 새 색상 반영 Store-->>LS: subscribe 콜백 → saveMemos() end rect rgb(255, 241, 241) Note over User, LS: 삭제 흐름 User->>Toolbar: 삭제 버튼 (✕) 클릭 Toolbar->>Actions: onDelete() Actions->>Store: deleteMemo(id) Store->>Store: memos에서 해당 id 제거 alt 삭제된 메모가 활성화 상태였을 때 Store->>Store: activeMemoId = null end Store-->>Canvas: memos 변경 → Memo 컴포넌트 언마운트 Store-->>LS: subscribe 콜백 → saveMemos() end

이미지 삽입 및 리사이즈
로컬 이미지를 커서 위치에 삽입하고, 호버 시 정렬·리사이즈 핸들을 제공한다
sequenceDiagram actor User as 사용자 participant Memo participant FileInput as <input type="file"> participant MemoContent participant Store as Zustand 스토어 rect rgb(237, 246, 255) Note over User, Store: 이미지 삽입 흐름 User->>Memo: 하단 이미지 버튼 클릭 Memo->>FileInput: fileInputRef.click() User->>FileInput: 로컬 이미지 파일 선택 FileInput->>Memo: onChange(file) Memo->>Memo: FileReader.readAsDataURL(file) Note over Memo: maxWidth = memo.width - 16
maxHeight = memo.height - 44 Memo->>MemoContent: insertImage(src, maxWidth, maxHeight) MemoContent->>MemoContent: <img> 생성 후 커서 위치에 삽입 MemoContent->>Store: onChange(div.innerHTML) end rect rgb(240, 255, 244) Note over User, Store: 이미지 정렬 · 리사이즈 흐름 User->>MemoContent: 이미지 호버 MemoContent->>MemoContent: showImgToolbar(img)
→ 정렬 툴바 + 리사이즈 핸들 표시 User->>MemoContent: 정렬 버튼 클릭 MemoContent->>MemoContent: applyAlignment() → img.style.margin 변경 MemoContent->>Store: onChange(div.innerHTML) User->>MemoContent: 우측 하단 투명 핸들 드래그 MemoContent->>MemoContent: document mousemove → img width/height 갱신

지도 삽입 및 마커 관리
Leaflet 지도를 메모 본문에 삽입하고, 마커 추가·삭제 모드를 제공한다
sequenceDiagram actor User as 사용자 participant Memo participant MemoContent participant Leaflet as Leaflet (initLeafletMap) participant Store as Zustand 스토어 rect rgb(237, 246, 255) Note over User, Store: 지도 삽입 흐름 User->>Memo: 하단 지도 버튼 클릭 Memo->>MemoContent: insertMap() MemoContent->>MemoContent: wrapper(data-map-wrapper) + container(data-map) 생성 MemoContent->>MemoContent: 커서 위치에 wrapper 삽입 MemoContent->>Leaflet: requestAnimationFrame → initLeafletMap(container) Leaflet->>Leaflet: L.map() + OpenStreetMap 타일 레이어 Leaflet->>Leaflet: wrapper 하단에 액션 바(마커 추가/삭제 버튼) 생성 MemoContent->>Store: onChange(getCleanedHTML) ← Leaflet DOM 제거 후 직렬화 end rect rgb(240, 255, 244) Note over User, Store: 마커 추가 흐름 User->>Leaflet: 마커 추가 버튼 클릭 Leaflet->>Leaflet: 입력 폼 표시 (위도/경도) User->>Leaflet: 위도·경도 입력 후 추가 Leaflet->>Leaflet: L.marker([lat, lng]).addTo(map) Leaflet->>Leaflet: container.dataset.markers 갱신 (JSON) Leaflet->>Store: onSave() → onChange(getCleanedHTML) end rect rgb(255, 241, 241) Note over User, Store: 마커 삭제 모드 흐름 User->>Leaflet: 마커 삭제 버튼 클릭 (토글 ON) Leaflet->>Leaflet: 모든 마커 → deletableIcon(빨간 ✕ 원형)으로 교체 User->>Leaflet: 특정 마커(✕ 아이콘) 클릭 Leaflet->>Leaflet: marker.remove() + markerList splice Leaflet->>Store: onSave() User->>Leaflet: 삭제 버튼 재클릭 (토글 OFF) Leaflet->>Leaflet: 모든 마커 → defaultIcon으로 원복 end

메모 리사이즈
우측 하단 핸들 드래그로 메모 크기를 조절하고 저장한다
sequenceDiagram actor User as 사용자 participant Handle as 리사이즈 핸들 participant useResizable as useResizable 훅 participant Store as Zustand 스토어 participant LS as localStorage 유틸 User->>Handle: pointerdown (드래그 시작) Handle->>useResizable: handlePointerDown(e) useResizable->>useResizable: startX, startY, startW, startH 기록 useResizable->>useResizable: pointermove 리스너 등록 loop 포인터 이동 중 User->>Handle: 포인터 이동 useResizable->>useResizable: newW = max(160, startW + dx)
newH = max(120, startH + dy) useResizable->>Store: resizeMemo(id, { width, height }) Store->>Store: size 업데이트, updatedAt 갱신 Note over Store: 매 이동마다 스토어 업데이트
(이미지 max 크기 재계산을 위해) end User->>Handle: pointerup (드래그 종료) useResizable->>useResizable: pointermove 리스너 해제 Store-->>LS: subscribe 콜백 → saveMemos()

전체 데이터 흐름 요약
모든 인터랙션은 동일한 단방향 파이프라인을 따른다
// ① 사용자 인터랙션 (이벤트 발생) 사용자 인터랙션 더블클릭(메모 생성) · 드래그(이동) · 타이핑(편집) · 버튼 클릭(색상·삭제) │ ▼ // ② 컴포넌트 이벤트 핸들러 (DOM 이벤트 수신) 컴포넌트 Canvas · Memo · MemoContent · MemoToolbar · ColorPicker │ ▼ // ③ 커스텀 훅 (UI 로직 ↔ 비즈니스 로직 분리 경계) 커스텀 훅 useCanvasInteraction · useMemoActions · usePersistence │ ▼ // ④ 스토어 액션 (단일 진실 소스) Zustand 스토어 createMemo · moveMemo · updateMemo · deleteMemo reorderToTop · setActiveMemo · hydrateFromStorage │ ├──────────────────────────────────────────────┐ ▼ ▼ // ⑤-A 구독 컴포넌트 리렌더 // ⑤-B localStorage 자동 저장 React 리렌더 subscribe 콜백 memos 배열을 구독하는 React 렌더 사이클 바깥에서 실행 컴포넌트만 선택적으로 리렌더 → saveMemos() → setItem()
